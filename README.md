# Wormhole
Using a relay server to build a virtual local network between different devices.

## Speficiations
1. All devices are set with the subnet of 172.30.1.0/24, device IPs are allocated statically following registration sequence. 
2. Each device have a unique ID, generated by the host when the client first join the network. This ID is coupled with the device IP in the subnet and do not change.
3. Every minute, online client send a packet to the host, to let the host know its current public IP and that it is online.
4. Proxy Content Format
```
| --- 4 bits --- | ----- other ----- |
|   status code  | original contents |
0 - OK, 1 - host don't exist, 2 - host down, 3 host unreachable
```

### Client
1. First thread: host -> tun -> udp socket -> remote server (port 4000)
2. Second thread: remote server -> udp socket -> tun -> host (port 4000)
3. Thrid thread: keep connection alive (port 4001)

### Server
1. First thread: forwarding udp packets (port 4000)
2. Second thread: keep connection alive (port 4001)
